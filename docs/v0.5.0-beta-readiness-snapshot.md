## Known Limits

- Not a fully automatic product manager
- Not a replacement for user confirmation
- Not a global reverse-engineering tool
- Not a GUI or hosted service
- Not a test runner replacement
- Not an automatic semantic analyzer
- Not a substitute for code review
- Migration tooling is policy-level, not command-level
- RPD Interview is guidance/skill-level, not deterministic CLI automation

## Not Yet v1.0

PBE is not v1.0 yet because:

- real-world adoption evidence is still limited
- artifact schemas may still evolve
- migration commands are not implemented
- package distribution is not stabilized
- RPD Interview is not a deterministic CLI engine
- public UX may still be rough
- breaking changes are still possible during beta

## Recommended Next Steps

1. Use PBE on one or more real external projects.
2. Collect dogfooding failures before adding new validators.
3. Promote only deterministic repeated failures to validator warnings/errors.
4. Evaluate `pbe doctor` / `pbe migrate` after more compatibility cases.
5. Avoid adding new Tree/State/Command unless complexity governance criteria are met.
