# UI/UX Spec

UI/UX Spec keeps screen behavior from being hidden inside generic task cards.

## Files

```text
.pbe/codex-execution-pack/05-ui-ux-spec.md
.pbe/codex-execution-pack/05-ui-ux-spec.json
.pbe/codex-execution-pack/15-ui-ux-evidence-checklist.md
```

## Required Screen Coverage

Each screen or UI flow should define:

- screen name
- purpose
- primary user
- primary flow
- required UI elements
- empty state
- loading state
- success state
- validation error state
- server error state
- permission state when applicable
- responsive priority
- accessibility expectations
- evidence required

## Evidence

For UI changes, Codex records implementation files, tests or manual verification notes, screenshot paths if available,
and accessibility notes.

If screenshots cannot be captured, Codex records the reason and a manual verification note.
