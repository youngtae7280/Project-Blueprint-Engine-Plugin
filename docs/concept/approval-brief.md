# Approval Brief

Status: outline only

## 1. Document Purpose

Approval Brief will define how PBE summarizes a human gate decision so the next stage can proceed without treating Codex
as the approver.

## 2. Scope This Document Will Cover

This future document should cover:

- what decision is being requested
- which Product, Project, Work, Test, Evidence, or Acceptance nodes are affected
- what the user is approving, revising, deferring, or rejecting
- what remains non-scope
- what downstream stage may start after approval

## 3. Currently Confirmed Decisions

- Only the user can accept product results.
- Human gates are required for product meaning, UI/UX, implementation scope, architecture runway, review result, and
  acceptance decisions.
- Approval text must not silently rewrite Product truth or accepted scope.

## 4. Still Undefined

- The exact Approval Brief format.
- Whether different gate types need different brief templates.
- How Approval Brief content should link to Acceptance Tree entries.
- How much CLI support is appropriate for approval summaries.

## 5. Next Phase Content

The next phase may define:

- gate-specific approval brief fields
- required node links
- examples for approve, revise, defer, block, and ask states
- compatibility behavior for existing review docs

Gate 4-6 outline gate only. This file does not define detailed approval policy or acceptance semantics.
