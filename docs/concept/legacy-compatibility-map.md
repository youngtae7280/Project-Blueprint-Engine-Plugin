# Legacy Compatibility Map

Status: outline only

## 1. Document Purpose

Legacy Compatibility Map will define how older PBE terms, files, and workflows map to the tree-native architecture
without reviving superseded product directions.

## 2. Scope This Document Will Cover

This future document should cover:

- RPD, WPD, VD, ACEP, and Revision compatibility mappings
- `.pbe/blueprint/*` compatibility views
- `.pbe/codex-execution-pack/*` compatibility package files
- legacy explanatory terms such as 5 Layer and Knowledge Graph
- migration guidance for older docs and examples

## 3. Currently Confirmed Decisions

- RPD means Product Tree growth.
- WPD means Project Tree and Work Tree derivation.
- VD means Test Tree derivation.
- ACEP means Cycle Contract and Node Execution Contract packaging.
- Revision means Change Tree, Impact Tree, and Reopen protocol.
- 5 Layer is a legacy explanatory frame, not active architecture.

## 4. Still Undefined

- Whether migration scripts are needed after documentation cleanup.
