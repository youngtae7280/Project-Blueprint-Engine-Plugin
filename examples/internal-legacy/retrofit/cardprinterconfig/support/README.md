# CardPrinterConfig Support Fixtures

Support fixtures preserve the user confirmation and candidate-match context
behind the formal retrofit records.

Current support files:

- `smart51-test-setting-confirmation-v0.json`
- `smart51-test-setting-candidate-match-v0.json`
- `smart51-test-setting-candidate-match-v0.md`
- `smart51-intake-view-v0.json`
- `cardprinterconfig-retrofit-graph-candidate-v0.json`
- `cardprinterconfig-retrofit-intake-v0.json`
- `retrofit-confirmation-protocol-v0.md`
- `smart51-setting-change-intake-v0.md`

They are not active change records by themselves. They explain how the
validated-then-reverted SMART-51 record reached its implementation decision.
